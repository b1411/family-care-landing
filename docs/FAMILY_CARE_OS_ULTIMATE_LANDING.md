## ФИЛОСОФИЯ

Каждый пиксель продаёт. Не словами — ощущением. Руководитель клиники должен за 30 секунд почувствовать: "это серьёзная компания, которая решит мою проблему". Мама должна за 15 секунд почувствовать: "здесь меня понимают и позаботятся". Инвестор должен почувствовать: "это не стартап из гаража, это инфраструктурный продукт".

**Три правила:**

1. Ни одного generic stock-вектора. Каждая иллюстрация уникальна и функциональна.
2. Ни одного слова, которое не ведёт к действию или пониманию.
3. Каждая анимация объясняет продукт, а не украшает страницу.

### Секция 1.1 — Hero: "Платформа"

**Текст:**

```
ЗАГОЛОВОК (Satoshi 900, --text-hero):
Каждая семья заслуживает
цифрового маршрута здоровья

ПОДЗАГОЛОВОК (Inter 400, --text-body-lg, --color-neutral-500):
Платформа сопровождения от зачатия до 2 лет —
для клиник, которые не теряют семьи после выписки

CTA 1 (primary filled): Запросить демо →
CTA 2 (outline): Как это работает ↓
```

**Почему этот текст:** "Каждая семья заслуживает" — эмоциональный вход. Не "мы делаем платформу", а "семья заслуживает". Подзаголовок сразу обозначает: для кого (клиники), что (сопровождение), зачем (не терять семьи). Два CTA: действие (демо) и информация (scroll).

**Визуал — 3D Floating Phone:**

Справа — 3D модель телефона (Three.js или CSS 3D transform). Внутри телефона — живой SVG-интерфейс.

Технически:

- Телефон: CSS perspective + rotateY(5deg) rotateX(-3deg)
- Mouse move: `useMouseInElement()` → subtle rotation ±5deg по осям (parallax feel)
- Внутри: SVG timeline с 5 событиями, рисуется при загрузке (GSAP timeline)
- Над телефоном: floating notification badge, покачивается (Lenis parallax speed: 0.8)
- Под телефоном: prescription pill card, parallax speed: 1.2
- За всем: gradient mesh background, анимированный (CSS hue-rotate 30s)

**Загрузка (sequence):**

```
0ms     — Gradient mesh background fade in
200ms   — Headline text: clip-path reveal line by line (clip-up)
600ms   — Subheadline: fade-up
800ms   — CTAs: fade-up with stagger
400ms   — Phone: slide-up from y:60 + opacity + 3D rotation settle
1200ms  — Phone screen lights up (white fill fade)
1400ms  — Timeline inside phone draws (stroke animation 1.2s)
1800ms  — Notification floats in from top
2000ms  — Prescription card slides in from bottom
2200ms  — Floating elements start their idle animations
```

**Metrics strip (под hero, overlapping):**

│  0→2 года  │  8 модулей │   24/7     │  <2 мин    │

### Секция 1.2 — Проблема: "Знакомая ситуация?"

**Подход: storytelling через два параллельных дня**

Не bullet-list болей. А два параллельных сценария: "День без платформы" vs "День с платформой". Читатель узнаёт себя.

**Layout:** Два столбца. Левый — розоватый tint (проблема). Правый — зеленоватый tint (решение). Между ними — вертикальная линия с анимированным переключателем.

**Левый столбец: "Без платформы"**

```
УТРО
📱 Мама листает 3 чата в поисках результата УЗИ
🤯 Не помнит: давала витамин D или нет
📞 Звонит в клинику узнать дату следующего приёма

ДЕНЬ
📋 Координатор вручную обзванивает 40 семей
📊 Руководитель не знает, сколько семей ушли после родов
👩‍⚕️ Педиатр на приёме: "А где ваши анализы?"

ВЕЧЕР
😰 Мама гуглит "красные пятна у ребёнка 4 месяца"
📝 Координатор закрывает Excel и уходит домой — 12 семей не дозвонились
```

**Правый столбец: "С платформой"**

```
УТРО
✅ Push: "Витамин D3 — 08:00" → одно нажатие → принято
📋 Все анализы в приложении, нашла за 5 секунд
🗓 Напоминание: "Педиатр завтра 10:00, заполните анкету"

ДЕНЬ
🎯 Координатор видит dashboard: 3 overdue, 5 вакцинаций на неделе
📈 Руководитель: retention 87%, конверсия в педиатрию 73%
👩‍⚕️ Педиатр открывает карточку: вся история, назначения, adherence 94%

ВЕЧЕР
🛡 Мама открывает Red Flag раздел → "Нормальная реакция на прививку"
✅ Координатор: все задачи закрыты, 0 overdue
😌 Мама: маршрут ребёнка на экране, следующий шаг понятен
```

**Анимация:**

- При загрузке секции: левый столбец виден, правый — скрыт (blur + opacity 0.3)
- При скролле: вертикальная линия-переключатель анимированно перемещается влево, правый столбец проявляется, левый уходит в blur
- Эффект: "было плохо → стало хорошо" — визуальная трансформация
- ScrollTrigger scrub: 0→0.5 = левый видим, 0.5→1 = правый видим

**SVG между столбцами:**

- Вертикальная линия с pulse-точкой посередине
- Точка = toggle: красная (проблема) → зелёная (решение) при переключении
- Иконка в точке морфится: ❌ → ✅

### Секция 1.4 — "Полный маршрут: от зачатия до 2 лет"

**Самая важная секция главной.** Показывает продукт в действии.

**Layout:** Горизонтальный scroll-driven timeline на полную ширину экрана.

**5 этапов с реальными данными:**

```
ЭТАП 1: ПЛАНИРОВАНИЕ И ЗАЧАТИЕ
Иконка: два кольца → соединение (морф)
─── Регистрация в платформе
─── Определение LMP, расчёт ПДР
─── Назначение врача-куратора
─── Первые назначения: фолиевая кислота

ЭТАП 2: БЕРЕМЕННОСТЬ (40 НЕДЕЛЬ)
Иконка: силуэт → растущий живот (морф)
─── 1-й скрининг (12 нед.) — УЗИ + биохимия
─── 2-й скрининг (20 нед.) — анатомия плода
─── ГТТ (26 нед.) — гестационный диабет
─── 3-й скрининг (33 нед.) — допплерометрия
─── Еженедельные приёмы (37-40 нед.)
─── 15+ анализов, 3 УЗИ, 10+ визитов

ЭТАП 3: РОДЫ И ПЕРВЫЕ ДНИ
Иконка: руки → ребёнок (морф)
─── Фиксация данных о родах
─── Создание профиля ребёнка (вес, рост, Apgar)
─── Запуск маршрута новорождённого
─── БЦЖ + Гепатит B (первые 24 часа)
─── Осмотр неонатолога
─── Патронаж на дому (1-я и 2-я неделя)

ЭТАП 4: ПЕРВЫЙ ГОД (0-12 МЕС.)
Иконка: лежит → сидит → ползёт (морф)
─── Ежемесячные осмотры педиатра
─── 10+ прививок по нацкалендарю
─── Невролог, ортопед, окулист (1, 3, 6, 9, 12 мес.)
─── УЗИ головного мозга, тазобедренных суставов
─── Витамин D3 ежедневно (365 напоминаний)
─── Прикорм, аллергены, рост, вес, milestones

ЭТАП 5: ВТОРОЙ ГОД (12-24 МЕС.)
Иконка: стоит → ходит (морф)
─── Осмотры каждые 3 месяца
─── Ревакцинация АКДС + ОПВ (18 мес.)
─── Стоматолог, логопед (18-24 мес.)
─── Оценка развития речи (24 мес.)
─── Итоговый check-up в 2 года
```

**Анимация:**

- Горизонтальная линия рисуется при вертикальном скролле (scroll-to-horizontal technique через GSAP ScrollTrigger pin + horizontal translateX)
- Каждый этап = экран шириной 100vw
- Иконка этапа морфится при входе в viewport (useMorph)
- События этапа appear с stagger при активации
- Прогресс-бар под timeline: заполняется по мере скролла
- Номер этапа: крупный Satoshi 900, opacity 0.05 → 0.1 при активации (watermark-эффект)

**Mobile:**

- Вертикальный scroll вместо horizontal
- Каждый этап — раскрываемый accordion
- Иконки морфятся при scroll-in (не при horizontal scroll)

**Формулы (прозрачные):**

- Семей уходит после родов: `families × 0.67` (67% отток — рыночные данные)
- С платформой остаётся: `lost × 0.70` (70% возвращаемость — целевой KPI)

- ROI = `(retained_revenue - platform_cost) / platform_cost × 100`

**Почему эти цифры работают:**

- 67% оттока — это conservative estimate, реальность часто хуже
- 1.8M ₸/семья/год — на основе публичных цен педиатрических пакетов Алматы
- Platform cost 1.5M/мес — реальный диапазон из нашей тарифной модели
